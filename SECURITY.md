# Security Policy

Supported versions
- Main branch. No versioned releases.

Reporting a vulnerability
- Please do not create a public issue for security problems.
- If GitHub Security Advisories are enabled for this repository, use "Report a vulnerability".
- Otherwise, email the maintainer privately or open a minimal public issue asking for a private contact channel.

Disclosure policy
- We will confirm receipt within 5 business days and provide a remediation plan if applicable.
- After a fix is available, we will credit reporters who wish to be acknowledged.

Best practices
- Do not include real secrets or personal data when filing issues or sharing sample data.
- Prefer environment variables or user secrets for any credentials during development.
